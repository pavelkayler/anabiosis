## Контекст
Этот файл является частью поэтапной реализации scanner-проекта на Bybit V5.

Работа по этому файлу должна вестись только после:
1. прочтения `0.md`;
2. прочтения последнего отчёта по предыдущему roadmap-файлу;
3. закрытия всех хвостов из предыдущего отчёта.

Если хвосты предыдущего шага не закрыты, реализацию этого файла начинать нельзя.

## Технические требования
- Код должен оставаться модульным и читабельным.
- Не допускается смешивать в одном большом файле unrelated logic из разных слоёв.

- Все решения, влияющие на scanner state и decision logic, должны быть объяснимыми и тестируемыми.

## Дополнительные smoke-check сценарии

Manual smoke checks должны включать:
- breakout preview inside candle, but no tradeable enter until 1m close;
- bias changed, setup missing → WAIT;
- geometry missing → WAIT;
- stale orderbook / stale liquidation scenarios;
- warmup not finished → no tradeable enter;
- trap vs failed continuation distinction on prepared replay case.


---

## Уточнения редакции v3

### Smoke-check должен покрывать
- bootstrap success;
- warmup progress;
- transition to `ANALYSIS_READY`;
- futures/spot flow updates;
- WAIT explanation correctness;
- INVALID explanation correctness;
- generation of tradeable decision только на закрытии 1m.

### Что фиксировать в smoke report
- фактическое время прогрева;
- были ли stale/degraded состояния;
- был ли хотя бы один preview/watchlist case;
- был ли хотя бы один valid decision case;
- совпадал ли current md file в header.


### Дополнительно проверить
- latency warning appears when ingest lag is forced;
- anomaly warning appears on malformed/duplicated replay payloads;
- degraded state clears after feed recovery.
