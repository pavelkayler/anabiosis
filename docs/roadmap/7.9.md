## Подробные задачи реализации
- Реализовать модуль, который переводит сырые OI/CVD/liquidation/book признаки в нормированные формы.
- Поддержать сразу три представления каждого ключевого признака: raw, volatility-normalized, z-score/percentile.
- Поддержать threshold profiles с версионированием и отдельным BTC profile preset.
- Подготовить DTO и persistence-поля для хранения normalized feature snapshots.
- Подготовить тесты на устойчивость расчётов на пустых, noisy и аномальных окнах.

---

## Специальные правила
- Не использовать только абсолютные пороги в decision logic.
- Не завязывать нормализацию только на mean/std; использовать robust rolling context.
- Разные symbol presets должны иметь возможность переопределять thresholds позже.

## Что считается завершением подэтапа
- Сервис нормализации существует и используется как отдельный слой, а не как разрозненные вспомогательные функции.
- Для BTC можно получить единый normalized feature snapshot.
- Threshold profile version сохраняется и может быть отдан наружу.

## Уточнения редакции v3

### Обязательные rolling windows для нормализации
- fast: `15m`
- medium: `1h`
- context: `24h`

### Обязательные методы
- robust z-score;
- percentile rank;
- volatility-normalized representation;
- raw representation.

### Запрет
Decision engine не должен принимать tradeable decision только на основе абсолютных значений метрик без local context normalization.
