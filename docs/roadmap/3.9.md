## Подробные задачи реализации
- Добавить системные сущности, позволяющие версионировать feature schema, regime rules, setup rules, trade geometry и explanation rules.
- Завести threshold profiles и bindings symbol → profile.
- Обеспечить запись версии правил в signal decision и validation outcomes.
- Гарантировать, что BTC profile preset существует как отдельный profile baseline.

## Что считается завершением подэтапа

---

## Уточнения редакции v3

### Где живут profiles и versions
- versioned config profiles хранятся в репозитории как source of truth;
- активный profile должен дублироваться в БД как runtime-resolved snapshot.

### Что обязательно версионировать
- feature schema;
- normalization rules;
- regime rules;
- bias rules;
- setup rules;
- trade geometry rules;
- explanation rules;
- BTC baseline profile.

### Правило
Ни одно signal decision и ни один validation outcome не должны существовать без явной ссылки на active rule/profile version set.
