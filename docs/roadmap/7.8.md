- Код должен оставаться модульным и читабельным.

## Дополнительные правила structure and levels

### Tradeable breakout
Tradeable breakout должен подтверждаться закрытием `1m` свечи за уровнем.

### Retest
Tradeable retest допустим только после подтверждённого breakout на закрытии `1m`.

### Приоритет structure
Structure levels являются primary source для:
- entry anchor;
- invalidation anchor;
- target zones.

Flow metrics не должны заменять отсутствие валидной геометрии.

### Trap
Trap setup требует именно structural fakeout:
- breakout состоялся;
- цена вернулась за уровень;
- уровень не reclaimed / не held;
- reverse flow подтверждает failure.

## Уточнения редакции v3

### HTF в BTC v1
Для decision logic HTF нужно считать как:
- `1h`
- `4h`

`1D` допускается как UI/context series, но не как обязательный trade driver в первой версии.

### Правило для retest
Tradeable retest возможен только после confirmed breakout на закрытой 1m свече.
Early retest внутри ещё формирующегося breakout считается только preview/watchlist состоянием.
