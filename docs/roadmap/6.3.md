## Уточнения редакции v3

### Формула futures CVD на ingestion-слое
На ingestion-слое нужно собирать quote-notional trade buckets:
- `buyQuote = Σ(size * price)` для taker buy;
- `sellQuote = Σ(size * price)` для taker sell.

### Обязательные поля
- raw quote delta;
- raw base delta;
- trade count;
- filtered notional delta;
- dedupe markers.

### Dedupe
Нужно использовать trade identity / seq / timestamp guards, чтобы один и тот же trade event не попадал в расчёт дважды.


## Уточнения редакции v5

### Data quality and anomaly handling
- входящие payloads должны проходить schema validation;
- битые/неполные сообщения не должны попадать в derived state;
- duplicate events должны быть отфильтрованы до записи в БД;
- невозможные значения (negative size, impossible price jump, broken timestamps) должны маркироваться как anomalies;
- anomaly не должна автоматически рушить процесс, но обязана логироваться и влиять на health/degraded markers.

### Latency tracking
- для каждого сообщения нужно хранить event time, receive time и processing time;
- должны считаться ingestion lag и processing lag;
- lag должен быть доступен для health layer и UI/debug panels.
