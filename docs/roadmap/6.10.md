## Обязательные правила
### Dedupe
- дедупликация по exchange trade id / seq, если поле доступно;
- если exchange id отсутствует, использовать composite identity: symbol + channel + eventTime + price + size + side;
- duplicate event не должен повторно менять derived state и не должен повторно писаться в БД.

### Anomaly classes
- broken payload;
- impossible timestamp;
- impossible price jump;
- impossible negative or zero-like sizes там, где они невозможны;
- out-of-order event beyond tolerated reorder window.

### Outlier policy
- anomaly event логируется;
- anomaly event не должен автоматически становиться частью primary analytics;
- при серии аномалий должен выставляться quality warning / degraded marker.

### Latency SLA tracking
Для каждого канала должны считаться:
- event lag = receiveTime - exchangeEventTime;
- processing lag = processedTime - receiveTime;
- end-to-end ingest lag = processedTime - exchangeEventTime.

### Базовые SLA для v1
- preferred end-to-end ingest lag для ticker/trades/orderbook: `< 1000ms`;
- warning: `>= 1000ms`;
- degraded candidate: `>= 3000ms` sustained.
